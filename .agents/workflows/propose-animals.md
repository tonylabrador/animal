# Workflow A: 动物寻猎与提案

> [!NOTE]
> 该工作流由用户手动触发，用于分析 `ANIMALS_LIST.md`，并在纵向深挖（科级）与横向拓宽（界门纲目级）双向维度上推荐 3~5 种极具科普价值和独特性的新物种。

## 执行步骤

1. 阅读当前 `ANIMALS_LIST.md`，理解目前数据库的收录情况。
    * **强制校验**：在推荐任何动物之前，必须绝对确认其为**种 (Species)** 级别，严禁推选亚种 (Subspecies)（如绝对不能推“家犬”或“家鸡”等亚种）。
    * **强制查重 (CRITICAL)**：在给出推荐列表之前，**必须**使用工具或仔细核对 `ANIMALS_LIST.md` 的内容，**绝对不允许推荐任何已经存在于该列表中的动物！**
2. **阶段 1：纵向深挖（丰满现有分类）**
    * 分析目前收录单薄（例如只有 1~2 种），但在现实生物学中极具多样性或科普价值的**科 (Family)**。
    * 为这些薄弱的科推荐 3~5 种极具代表性、外观奇特或有故事性的具体**物种 (Species)**。
3. **阶段 2：横向拓宽（填补大类空白）**
    * 评估整个数据库在界门纲目维度的平衡性。
    * 找出目前极度匮乏的**大类（纲 Class 或 目 Order）**。
    * 推荐 3~5 种能够作为该全新大类「门面担当」的旗舰物种。
4. **人机交互：请求用户审核**
    * 向用户完整且生动地打印出两个阶段的推荐列表及推荐理由。
    * 询问用户希望选取列表中的哪几个动物加入数据库。
    * **强制暂停：使用 `notify_user` 等待用户的明确指示。**
5. **记录决策**
    * 在收到用户的选择后，将选中动物的「中文名、英文名、学名」以列表或表格形式追加写入 `ANIMAL_TO_ADD.md` 文件的尾部。
    * 告知用户本工作流结束，并提示他们可以随时运行 Workflow B (`generate-and-qa.md`) 来生成数据。
