# Workflow B: 数据制造与严格质检

> [!NOTE]
> 该工作流根据 `ANIMAL_TO_ADD.md` 中的清单，严格依照全局规范生成高质量 JSON 并写入系统，自带闭环 QA。

## 执行步骤

1. 阅读 `ANIMAL_TO_ADD.md`，提取待生成的动物列表。如果文件为空，则终止流程并通知用户。
2. **执行逻辑 (Agent 2 - 生成)**
    * 针对每一个待加入的动物，按 `formatting.md` 中的要求搜集对应的百科信息。
    * 内容生成必须生动有趣、包含硬核特征、中英双语、且符合该动物所属生境的实际情况。
    * **严格生成高精度多边形坐标**，并在 JSON 产出时强制对 `global_distribution_polygons` 数组进行**横向压缩排版**（one polygon per line）。
3. **执行逻辑 (Agent 3 - 严格 QA)**
    * 暂存并拦截刚刚生成的 JSON 内容，先进行自我审查。检查清单：
        - [ ] 是否完全采用了种级单词 (Species) 且不包含亚种？
        - [ ] 所有的长短文本的中英双语是否完整？
        - [ ] 中文语境下是否 100% 采用了直角引号「」？
        - [ ] 分布多边形的经纬度节点是否具备高精度细节（绝不能是简单的 4 个点大方块）？
        - [ ] 分布多边形在 JSON 代码中是否横向排版在同一行？
    * 若有任何不合格项，内部自行打回并修正，直到100%符合标准。
4. **执行逻辑 (Generator - 下载与列表脚本)**
    * 暂存通过文本 QA 的 JSON 到 `data/animals/<id>.json`。
    * 执行脚本 `node download_images.js`，让爬虫自动为新动物抓取对应的真实照片到本地。
    * 执行脚本 `node update_animals_list.js`，自动将新动物追加并刷新主目录的 `ANIMALS_LIST.md`。
5. **执行逻辑 (Reviewer - AI 视觉质检)**
    * **核心拦截**：读取刚刚被下载保存下来的图片文件，利用 AI 视觉能力进行严格的验证判断。
    * **质检标准**：爬下来的图片是否真的是该动物？（如：是野生动物的形态，而不是漫威人物、Logo、卡通画或毫不相干的风景）。
    * **报警机制**：如果发现“图文严重不符”或有极强的误导性，必须**使用 Alert 强提示** 向用户报警，告知具体是哪只动物的图片抓错了，建议其手动前往调整。
6. **执行逻辑 (自动更新分类树)**
    * 执行脚本 `node build_taxonomy_tree.js`，自动根据最新的 `data/animals/*.json` 重建 `data/taxonomy-tree.json`，确保 Taxonomy Explorer 页面即时同步最新数据。
7. **输出与收尾操作**
    * 从 `ANIMAL_TO_ADD.md` 中彻底删除本次成功处理完毕的所有动物记录。
    * 总结本次工作流的执行情况（包括写入了几条记录、是否有图片被 Alert），向用户汇报执行完毕。
